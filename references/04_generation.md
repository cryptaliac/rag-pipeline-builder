# Step 4: Generation

The goal of this step is to use the relevant document chunks (context) retrieved in [Step 3](03_retrieval.md) to generate a final, coherent answer to the user's query. This corresponds to the `LLM Generation Layer` in the diagram.

## Core Process

1.  **Define a Prompt Template**: Create a template that will structure the information (context and query) passed to the LLM.
2.  **Set up the LLM (Large Language Model)**: Specify the LLM that will be used for answer generation.
3.  **Generate Response and Post-process**: Get the response from the LLM, refine it, and include source information.

## 1. Defining a Prompt Template

The prompt template is one of the most critical factors affecting the performance of a RAG system. The key is to instruct the LLM to generate answers based *only* on the provided context.

**Example: Prompt template in LlamaIndex**

```python
from llama_index.core import PromptTemplate

# A QA (Question-Answering) prompt template
# {context_str} will be filled with the document chunks retrieved in Step 3.
# {query_str} will be the user's original query.

qa_prompt_tmpl_str = (
    "Context information is below.\n"
    "---------------------\n"
    "{context_str}\n"
    "---------------------\n"
    "Given the context information and not prior knowledge, "
    "answer the query. If the context does not provide an answer, "
    "just say that you couldn't find an answer in the provided documents.\n"
    "Query: {query_str}\n"
    "Answer: "
)
qa_template = PromptTemplate(qa_prompt_tmpl_str)

# Update the query engine with the new template.
# Assume query_engine was created in Step 3.
query_engine.update_prompts({"response_synthesis:text_qa_template": qa_template})
```

> **[Tip]** Explicitly telling the LLM not to use "prior knowledge" and defining its behavior when the context lacks an answer is highly effective in reducing hallucinations.

## 2. Setting Up the LLM

The diagram shows a setup where the LLM is run locally using `Ollama`. `Ollama` is a tool that makes it easy to install and run various open-source LLMs like Llama 3 and Mistral. Alternatively, you can use API-based models like OpenAI's GPT.

## 3. Generating and Post-processing the Response

Everything is now set up. When you pass a query to the engine, the entire process of **Retrieval -> Prompting -> Generation** is executed internally.

```python
# Use the hybrid search engine configured in Step 3.
response = query_engine.query("What is RAG?")

# The answer text generated by the LLM
print(response.response)

# Check the metadata of the source nodes (documents) used for the answer.
# This allows you to provide a "View Sources" feature to the user.
for node in response.source_nodes:
    print(f"Source: {node.metadata.get('file_name')}, Page: {node.metadata.get('page_label')}")
```
